1. Computer Systems and Data Processing

(a) Mainframe vs Microcomputer (1 mark)

- Mainframe Computer: Large, powerful systems used by organizations for bulk data processing and critical applications.
- Microcomputer: Smaller, personal-use computers like desktops or laptops, designed for individual tasks.

(b) Data Processing Modes

- Transaction Processing: Real-time processing of individual transactions (e.g., ATM withdrawals).
- Batch Processing: Accumulated data is processed in groups at scheduled times (e.g., payroll systems).

(c)(i) Properties of a Relation in Databases

- Uniqueness: Each row (tuple) is unique.
- Atomicity: Each cell contains a single value.
- No Ordering: Rows and columns have no inherent order.
- Consistency: All entries in a column are of the same data type.

(c)(ii) Why Binary is Used in Computers

- Simplicity of Hardware: Binary (0s and 1s) aligns with electrical states (on/off).
- Reliability: Easier to detect and correct errors in binary systems.

2. Database Concepts and Audit

(a) Composite Key vs Super Key

- Composite Key: A key made of two or more attributes that uniquely identify a record. - Super Key: Any combination of attributes that uniquely identifies a record, including composite and candidate keys.

(b) Functions of an Auditor in Database Audit (2 marks)

- Verify Data Integrity: Ensure data is accurate and consistent.
- Review Access Controls: Check who has access and how it's managed.

- Master File
- Description: Stores permanent data that rarely changes.

- Transaction File

3(a) Four Fault Tolerance Techniques

- Redundancy: Using backup components like extra hard drives.
- RAID (Redundant Array of Independent Disks): Combines multiple disks to prevent data loss.
- Uninterruptible Power Supply (UPS): Maintains power during outages.
- Data Backup: Regularly saving copies of data to restore after failure.

3(b) Database Terms

(i) Transaction:

- A sequence of database operations treated as a single unit, which must be completed fully or not at all.

(ii) Referential Integrity:

- Ensures relationships between tables remain consistent; for example, a foreign key must match a valid primary key in another table.

3(c) Two Advantages of Direct Data Input

- Accuracy: Reduces human error since data is entered directly from the source.
- Speed: Faster than manual entry, especially with devices like barcode scanners or sensors.

(d)(i) Three Functions of an Information System in an Organization

An information system serves multiple roles. Here are three key functions:
- Data Processing and Management
- Collects, stores, processes, and retrieves data to support decision-making.
- Example: A payroll system calculates employee salaries based on attendance and tax data.
- Communication and Coordination
- Facilitates internal and external communication across departments.
- Example: Email systems, shared databases, and collaborative platforms like intranets.
- Decision Support
- Provides analytical tools and reports to guide strategic planning.
- Example: Sales dashboards help managers identify trends and adjust marketing strategies.

4(b) Logical vs Physical Data Independence

Definitions:

- Logical Data Independence:

Ability to change the logical schema (e.g., adding new fields) without altering the application programs.

- Physical Data Independence:

Ability to change the physical storage (e.g., indexing, file formats) without affecting the logical schema or applications.
